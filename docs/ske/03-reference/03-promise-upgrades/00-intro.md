When you have updated a Promise, you can manage the process of updating Resource Requests via Upgrade Plans and Upgrade Runs. Upgrade Plans define the strategy for how an upgrade
will be performed whilst Upgrade Runs detail the execution of a given Upgrade Plan.

- [Upgrade Plans](./upgrade-plans)
- [Upgrade Runs](./upgrade-runs)

For task-based walkthroughs of common upgrade strategies, such as canary rollouts, maintenance windows, and scheduled
upgrades, see the [Managing Promise Upgrades](/ske/guides/promise-upgrades) guide.
