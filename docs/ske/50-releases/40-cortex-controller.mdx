## Overview

The SKE Cortex Controller allows you to automate the process of populating the Cortex catalog with Entities as well as generate Workflows that can be used to provision services backed by Kratix Promises. For the Cortex Configuration see documentation [here](/ske/integrations/cortex/intro).

The release manifest is available at:

```
http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-cortex-controller/
```

## Release Notes

### [v0.8.0](http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-cortex-controller/v0.8.0/) (2026-06-02)

#### Features

* The [visibility of the namespace field in the Cortex UI](/ske/10-integrations/11-cortex/02-generating-cortex-components#customising-name-and-namespace-fields) can now be hidden and request deployed to a fixed namespace. The new `kratix.io/cortex-fixed-namespace` annotation makes it possible to set fixed namespace to deploy resources requests of promises to. When set, the `metadata.namespace`
field is hidden in the workflow in the Cortex UI.

#### Security Fixes

* Security updates

### [v0.7.0](http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-cortex-controller/v0.7.0/) (2026-05-20)

#### Features

* Allow [customising the display name and description](/ske/integrations/cortex/generating-cortex-components#customising-name-and-namespace-fields) of the Name and Namespace fields in generated Cortex workflows via Promise annotations.

### [v0.6.0](http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-cortex-controller/v0.6.0/) (2026-05-15)

#### Features

* Create global Cortex resources when the SKE Cortex Controller starts to allow allows all Kratix-managed objects (Promises and resource requests) to be surfaced together under a single catalogue item.
* Allow skipping creation of Cortex workflows via annotations on Promises.

* Security updates
