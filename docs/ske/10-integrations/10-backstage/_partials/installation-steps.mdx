1. [**Create the Entity State Store**](/ske/integrations/backstage/installation/creating-entity-state-store) — Configure a Kratix State Store and Destination so Kratix knows where to write Backstage entity files. Backstage and Kratix must point at the same store.

1. [**Configure Catalogue Ingestion**](/ske/integrations/backstage/installation/configuring-catalogue-ingestion) — Configure Backstage to read its Software Catalog from the State Store that Kratix writes to.

1. [**Install and Configure the Plugins**](/ske/integrations/backstage/installation/plugins) — Install the `@syntasso/plugin-ske-frontend` and `@syntasso/plugin-ske-backend` packages into your Backstage app. The frontend plugin renders dedicated pages for Kratix Promises and Resources; the backend plugin handles Resource Requests and catalog queries.

1. [**Install the Backstage Controller**](/ske/integrations/backstage/installation/generating-backstage-components) — Deploy the Backstage Controller via the SKE Operator to automatically generate Backstage Components and Templates from your Kratix Promises.
