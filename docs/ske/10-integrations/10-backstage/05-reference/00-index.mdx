:::note

Most platforms should generate Backstage entities with the [Portal Controller](/ske/integrations/portal-controller/intro) rather than wiring the generator into pipelines by hand.
The standalone Backstage Controller continues to work for at least one more release, see
[Migrating from the Backstage Controller](/ske/integrations/portal-controller/migrating-from-backstage).

:::

Reference documentation for the SKE Backstage integration.

- [**SKE Backstage Generator**](./generator) — Automatically populate your Backstage catalog from Promises using the SKE Backstage Generator.
- [**YAML File Format**](./yaml-file-format) — The entity YAML format expected by the SKE plugins for Promises and Resource Requests.
- [**SKE Component Promise**](./generator-promise) — Use the SKE Component Promise to manage Backstage components as a platform service.
- [**SKE Backend Plugin**](./ske-backend-plugin) — Actions and endpoints shipped by the `ske-backend` Backstage plugin.
