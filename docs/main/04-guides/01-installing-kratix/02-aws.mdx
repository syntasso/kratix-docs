## Set up Platform Cluster {#platform-setup}

If you are not using a pre-existing cluster, create your Platform Cluster by
following the [EKS getting started docs](https://docs.aws.amazon.com/eks/latest/userguide/getting-started.html) 

Once completed, log in to your cluster using the `aws` cli and set the `PLATFORM` environment
variable:

```bash
aws eks update-kubeconfig --region <region-code> --name <cluster-name>
kubectl config current-context
export PLATFORM=<platform-context-name>
```

### Configure a Destination and State Store

Once Kratix is installed, you will want to configure a Destination it can schedule your
workflow outputs to and a State Store to store that Destination's content.

For generic instructions, please visit the [State Store](https://docs.kratix.io/main/reference/statestore/intro)
and [Destination](https://docs.kratix.io/main/reference/destinations/intro) pages.

To use a AWS S3 Bucket as your State Store, you can refer to the [Kratix Examples repository](https://github.com/syntasso/kratix-examples) which provides easy
install scripts.
