SKE-Operator

Overview​

The SKE-Operator is the operator for managing the installation and upgrading of SKE. For installation documentation see here.

The release manifest is available at:

http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-operator/

The helm chart is available at:

http://syntasso-enterprise-releases.s3-website.eu-west-2.amazonaws.com/#ske-operator-helm-chart/

To see which version of the helm chart corresponds to which version of the SKE-Operator, please refer to the release notes below.

Release Notes​

v0.19.0 (2026-01-12)​

Released in Helm Chart version version 0.52.0

Features​

  • This release adds the functionality to deploy the Backstage Controller via the SKE Operator.

v0.18.3 (2025-12-16)​

Released in Helm Chart version version 0.51.0

Maintenance​

  • Security updates

v0.18.2 (2025-11-21)​

Released in Helm Chart version version 0.50.0

Bug fixes​

  • Use the correct pipeline adapter image when Kratix version is set to 'latest'.

Known issues​

  • There is a known security vulnerability associated with an indirect dependency of SKE Operator. SKE Operator is not impacted by the vulnerability. The upstream dependency will be updated in the next release.

v0.18.1 (2025-10-14)​

Released in Helm Chart version version 0.48.0

Features​

  • Support for ensuring the proper Pipeline Adapter ConfigMap is applied during upgrades.

Known issues​

  • There is a known security vulnerability associated with an indirect dependency of SKE Operator. SKE Operator is not impacted by the vulnerability. We are waiting for an upstream dependency release to update in the next version.

v0.17.0 (2025-10-07)​

Released in Helm Chart version version 0.47.0

Features​

  • Support for the Pipeline Adapter ConfigMap for SKE v0.37.0 or later.

Helm Chart v0.46.1(2025-09-24)​

Fixes​

  • Image Pull Secret will not be correctly interpolated into the post-deploy jobs.

v0.16.1 (2025-08-22)​

Released in Helm Chart version version 0.46.0

Features​

  • This release consolidates the images that are required when deploying SKE with the SKE Operator Helm Chart. The operator image can be used to run the Helm Chart's post-install jobs, eliminating the dependency on the kratix-pipeline-utility image.

v0.15.0 (2025-08-06)​

Released in Helm Chart version version 0.45.0

Features​

  • You can configure custom CA issuer used for SKE certificates using ske-operator.

v0.14.0 (2025-07-16)​

Released in Helm Chart version version 0.43.0

Features​

  • The SKE Operator will now iterate over the available versions when upgrading SKE. For example, if your installation is on version v0.29.0, and you set the version to v0.31.0, the SKE Operator will first upgrade to v0.30.0 and then to v0.31.0. This ensures a smooth upgrade process.
  • Resource requests and limits of CPU and memory can now be configured for the SKE deployment.

v0.13.0 (2025-07-11)​

Released in Helm Chart version version 0.42.0

Features​

  • In rare occasions where a release version is not available, the SKE Operator can now handle upgrades to the next appropriate version. The version is determined by the release-metadata.json file within the release artefact.

Helm Chart v0.42.0(2025-07-11)​

Features​

  • Resource limits, nodeSelector, tolerations and affinities can now be configured for the SKE Operator deployment. Check the values.yaml for more information.

Helm Chart v0.39.0 (2025-05-25)​

Features​
  • Deleting the SKE Operator by uninstalling the Helm Chart will no longer result in the deletion of your SKE instances, CRDs or the registry secret. To allow for the deletion of these resources when uninstalling the SKE Operator via the chart, the value global.skeDeployment.deleteOnUninstall has been introduced.

v0.12.1 (2025-05-11)​

Released in Helm Chart version version 0.38.0

Maintenance​

  • Security updates

v0.11.0 (2025-04-16)​

Released in Helm Chart version version 0.35.0

Features​

  • Bump default ske-operator container resources

Maintenance​

  • Bump to Go 1.23
  • Bump dependencies

v0.10.1 (2025-01-05)​

Released in Helm Chart version version 0.32.0

Features​
  • This release removes the pre and post upgrade checks that take place during a Kratix upgrade
Chart features​
  • A syntasso-registry secret is no longer created if an imageRegistry.imagePullSecret is specified

v0.10.0 (2024-12-19)​

Released in Helm Chart version version 0.30.0

Features​
  • This release renames the kratix-platform-wc-img-config configmap to kratix-platform-pipeline-adapter

v0.9.0 (2024-12-12)​

Released in Helm Chart version version 0.29.0

Note: This version of the SKE Operator is only compatible with SKE versions v0.8.0 or above.

Features​

  • Enhanced support for Kratix upgrades, including post-upgrade checks written to the Kratix status

Helm Chart v0.26.0 (2024-11-26)​

Deploys SKE Operator version v0.6.0

Features​

  • This chart provides support for provisioning additional SKE and Kubernetes resources alongside the SKE Operator and SKE Deployment. With the skeDeployment.additionalResources, property, users can specify any additional resources that rely on the existence of a SKE Deployment (e.g. Destinations, BucketStateStore) whilst the higher level additionalResources property can be used to specify any other Kubernetes resources required (e.g. Secrets).

v0.6.0 (2024-11-21)​

Released in Helm Chart version 0.25.0

Bug Fixes​

  • Fix bug to ensure Kratix Deployment readiness is properly checked before being marked as Available

v0.5.0 (2024-11-12)​

Released in Helm Chart version 0.22.0

warning

This version of the Helm chart introduces a breaking change in how it manages the Kratix CRD. To ensure a smooth upgrade, you must run the following manual commands before upgrading:

kubectl --context=$PLATFORM label crd kratixes.platform.syntasso.io app.kubernetes.io/managed-by=Helm
kubectl --context=$PLATFORM annotate crd kratixes.platform.syntasso.io meta.helm.sh/release-name=ske-operator meta.helm.sh/release-namespace=kratix-platform-system

These steps are required to align the CRD management with the new Helm chart structure. Failing to perform this migration will result in upgrade failures.

  • Support installing SKE Operator and SKE without cert-manager

Released in Helm Chart version 0.21.0

  • Update default image host to registry.syntasso.io

v0.4.0 (2024-10-17)​

Released in Helm Chart version 0.19.0

Bug Fixes​

  • Fix bug that made the Operator version configurable as this resulted in errors during upgrades

v0.3.1 (2024-10-15)​

Released in Helm Chart version 0.17.0

Bug Fixes​

  • Fix bug preventing Kratix resources from being updated gracefully

v0.3.0 (2024-10-10)​

Released in Helm Chart version 0.16.0

Features​

  • Support pulling releases from a Git Repo for air-gapped environments

Bug Fixes​

  • Fix bug preventing correct version of SKE being returned from the Operator
